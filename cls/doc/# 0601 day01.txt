# 0601 day01

Spring
    ==> MVC 패턴을 기반으로 하는 웹프로그램 개발을 위해서
        좀 더 다양한 기능을 제공하기 위한 프레임워크의 일종이다

        즉, 우리가 지금까지 직접 만들어서 사용했던
        'dispatch servlet' 부터 모든 작업을 미리 만들어 놓고
        이용할 수 있도록 해놓은 하나의 개발 툴이다

        전자정부 3.0 에서 지원하는 프레임워크의 일종이다
    
    특징]
        1. 요청이 컨트롤러 단위가 아니고 함수단위이다
            
            우리가 지금까지 하나의 요청이 생기면
            하나의 컨트롤러를 제작하고 작업을 했어야 했다
            하지만 스프링은 컨트롤러 클래스는 한 개만 만들고
            그 내부에 여러 개의 함수를 이용해서 요청을 처리하도록 한다

            하나의 클래스 안에 여러 개의 함수를 만드는 것은
            각각의 함수의 공통기능을 따로 함수로 만들어서
            공통적으로 이용할 수 있도록 할 수 있다

        *
        2. 빈(Bean)처리를 할 수 있다
            (객체를 만들어주는 처리)

            사용할 클래스를 한 곳에 모아서 new 처리를 할 수 있다
            (스프링에서는 직접 new 시키는 경우가 매우 드묾)

            소스 관리가 매우 간편해진다

        ***
        3. DI 기법을 이용할 수 있다
            우리말로 번역하자면 '느슨한 주입'이라고 하며
            빈처리에서 new 시킨 클래스를 자동으로 사용하는 기법

        ***
        4. AOP 기법을 이용할 수 있다
            우리말로 번역하자면 '전처리 클래스'라고 하며
            특정 작업이 실행되는 순간 필요한 다른 작업을 우선 처리하는 기법
            (ex. 로그인페이지에 들어가기 전, 현재 세션에 ID가 있는지 확인하는 등)

            로그인 검사는 거의 모든 작업에서 필요한 작업이었다
            (공통적인 작업이 매번 코딩되어야 했다)

이클립스에 스프링 프로젝트 설치
    1. ADT 방식
        ==> 이클립스 자체를 처음부터 스프링 프로젝트에 맞도록 만들어놓고
            필요한 라이브러리를 하나씩 복사해서 프로젝트를 진행하는 방법

    2. Maven 방식
        ==> 이클립스 자체는 원상태를 사용하면서
            Maven 서버를 이용해서 필요한 라이브러리를 자동으로 불러와서 사용하는 방법
            (매번 Maven 서버에 연결함)

    # 우리는 메이븐 방식으로 처리하도록 한다
        단점: 항상 네트워크가 개설되어야 한다(인터넷 연결 확인)

    작업순서
        1. Maven 서버 설치
            사이트: http://maven.apache.org/download.cgi

        2. 관리하기 편한 위치에 압축을 푼다

        3. 시스템 환경변수에 Maven 서버를 등록한다
            Path에 Maven의 실행파일들의 위치를 추가한다

        4. 설치확인
            (cmd에서)
            mvn --version
